About BackBorn

What is BackBorn?

Overview

This section gives you a quick look at what BackBorn is about. Don't worry, it's not some heavy documentation page—just a casual intro.
BackBorn is a project I started on June 9th, 2025. It's my very first backend and full-stack project, and it's still a work in progress.
If you don't know me, which you surely don't, I'm Astrael—a self-taught web developer who enjoys learning by doing and building stuff from scratch.
With this project, I wanted to dive into backend development using Node.js, Express, and SQLite. It's my way of understanding how things really work behind the scenes.
I'm also learning JavaScript more seriously, experimenting with Python, and occasionally messing around with a bit of C++ just for fun.
BackBorn is where I can track my progress and try new things without pressure. I've really enjoyed working on it so far.
If you're curious about what I've done or want to see how this project evolves, feel free to check out my GitHub!

GitHub

Frontend

The frontend of BackBorn is built with HTML, CSS, and JavaScript. It's designed to be simple and user-friendly, focusing on functionality rather than flashy design.
I wanted to create a clean and intuitive interface that makes it easy for users to navigate and interact with the application.
The frontend is responsive, so it works well on both desktop and mobile devices. I've used some basic CSS techniques to ensure a consistent look and feel across different screen sizes.
If you're interested in the code, you can find it on the official GitHub repository of the project.

Frontend repository

Backend

The backend of BackBorn is built using Node.js and Express. I've chosen SQLite as the database for its simplicity and ease of use.
The backend handles user authentication, data storage, and all the logic needed to make the application work.
I've implemented RESTful APIs to allow the frontend to communicate with the backend seamlessly.
This project has been a great learning experience for me, and I'm excited to continue improving it as I learn more about backend development.
If you're curious about the backend code, you can find it on the official GitHub repository of the project.
